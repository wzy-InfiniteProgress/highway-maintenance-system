## 📋 测试步骤

### 1. 登录测试
1. 访问 http://localhost:3000/login
2. 使用测试账户登录（建议先使用admin）
3. 验证登录成功后跳转到首页

### 2. 首页测试
1. 查看统计卡片（维护心得数、用户数等）
2. 测试快速操作卡片
3. 点击"查看维护心得"进入帖子列表

### 3. 维护心得列表测试
1. 查看帖子列表（应显示3个示例帖子）
2. 测试搜索功能（输入关键词搜索）
3. 测试分类筛选（选择不同分类）
4. 测试分页功能
5. 点击"创建心得"按钮

### 4. 创建帖子测试
1. 填写标题（必填）
2. 选择分类（必填）
3. 输入内容（必填）
4. 添加标签（可选）
5. 设置公开/私有
6. 点击"创建"按钮
7. 验证创建成功并跳转到列表页

### 5. 帖子详情测试
1. 点击任意帖子查看详情
2. 验证浏览次数增加
3. 查看作者信息
4. 查看标签显示
5. 测试编辑按钮（如果是作者或管理员）
6. 测试删除按钮（如果是作者或管理员）

### 6. 编辑帖子测试
1. 点击帖子详情页的"编辑"按钮
2. 修改标题、内容或分类
3. 点击"更新"按钮
4. 验证更新成功并跳转到详情页

### 7. 权限测试
1. 使用viewer账户登录
2. 尝试创建帖子（应该成功）
3. 创建私有帖子
4. 退出登录
5. 使用admin账户登录
6. 查看所有帖子（包括私有帖子）
7. 尝试编辑viewer的帖子（应该可以，因为是管理员）

### 8. 登出测试
1. 点击右上角用户名
2. 点击"退出登录"
3. 验证跳转到登录页

## 🔧 技术栈

- **前端框架**: Next.js 14 (App Router)
- **UI组件库**: Ant Design 5
- **样式方案**: Tailwind CSS 4
- **语言**: TypeScript 5
- **数据库**: SQLite (开发环境）
- **认证**: JWT + bcrypt
- **状态管理**: React Hooks

## 📊 数据库设计

### User表
- id: 用户ID（UUID）
- username: 用户名（唯一）
- email: 邮箱（唯一）
- password: 密码（bcrypt加密）
- role: 角色（admin/maintainer/viewer）
- department: 部门
- isActive: 是否激活
- createdAt/updatedAt: 时间戳

### Post表
- id: 帖子ID（UUID）
- title: 标题
- content: 内容
- category: 分类
- tags: 标签（JSON数组）
- authorId: 作者ID（外键）
- isPublic: 是否公开
- views: 浏览次数
- createdAt/updatedAt: 时间戳

### Category表
- id: 分类ID（自增）
- name: 分类名称（唯一）
- description: 分类描述
- createdAt: 创建时间

## 💡 使用建议

1. **开发环境**: 当前使用SQLite数据库，适合开发测试
2. **生产环境**: 建议使用PostgreSQL或MySQL
3. **部署**: 可以部署到Vercel、Netlify等平台
4. **安全性**: 生产环境务必修改JWT_SECRET
5. **性能**: 大数据量时考虑添加缓存和索引
